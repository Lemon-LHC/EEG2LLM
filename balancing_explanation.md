# 睡眠阶段分类的平衡策略原理与效果

## 你的数据集特点分析

你的数据集呈现典型的"四高两低"分布特征：
- **主要类别**：W (21.6%)、N1 (19.9%)、N2 (21.6%)、R (23.5%)相对均衡
- **少数类别**：N3 (9%)和N4 (4.4%)明显偏低

这种分布反映了真实世界中睡眠阶段的自然分布，即深度睡眠(N3/N4)通常占比较少。然而，从机器学习角度看，这种不平衡可能导致模型对N3和N4识别能力较弱，需要通过特定策略改善。

## 推荐平衡策略的原理

### 1. 为什么选择balanced策略？

**原理**：balanced策略通过过采样少数类(N3/N4)来平衡物理样本数量。

实现细节：
```python
# 根据平衡系数计算目标样本数量
target = int(median_count + balance_alpha * (max_count - median_count))

# 对于N3和N4，当前样本数<目标数量时，进行过采样
if current_count < target_count:
    # 计算需要额外添加的样本数
    extra_count = target_count - current_count
    # 随机抽样添加
    for _ in range(extra_count):
        # 随机选择一个样本复制
        balanced_data.append(random.choice(samples))
```

**选择理由**：
1. 你的N3和N4样本数量明显偏低，需要直接增加物理样本数
2. balanced策略可有效增加少数类样本，同时保持其他类别样本不变
3. 适中的balance_alpha=0.5可避免过度过采样带来的过拟合风险

### 2. 为什么选择sqrt_inverse权重方法？

**原理**：sqrt_inverse权重方法通过平方根函数缓和了反比例权重的极端效果。

数学表示：
```
W(c) = √(max_count / count(c))
```

**与其他方法对比**:
- **inverse**: W(c) = max_count / count(c)
  - 对于N4(4.4%)，权重约为5.34，过于极端
- **sqrt_inverse**: W(c) = √(max_count / count(c))
  - 对于N4(4.4%)，权重约为2.31，更为适中
- **effective_samples**: 需要额外调整beta参数，增加复杂性

**选择理由**：
1. sqrt_inverse提供温和但有效的平衡效果
2. 避免过度补偿小类别导致的训练不稳定
3. 计算简单，不需要额外的超参数调整

## 平衡后对训练的影响

### 数据层面影响
1. **物理样本分布变化**：
   - N3从9%增加到约15.5%
   - N4从4.4%增加到约15.5%
   - 其他类别比例相应下降，但样本数不变

2. **样本权重分布**：
   - N3权重约为1.54
   - N4权重约为2.20
   - 其他类别权重接近1.0

### 模型训练影响
1. **梯度更新**：对N3和N4样本产生的梯度给予更高权重，加速模型对这些类别的学习
2. **决策边界**：更倾向于识别N3和N4，提高对少数类的召回率
3. **损失函数**：N3和N4样本的损失被放大，促使模型更加关注这些困难样本

### 预期效果
1. **整体准确率**：可能略有下降（因为原始准确率往往偏向多数类）
2. **F1分数**：应显著提高（特别是宏平均F1，反映全类别平均性能）
3. **少数类召回率**：N3和N4的召回率应有明显提升
4. **平衡度**：各类别的准确率差异应该减小

## 如何验证效果

在训练完成后，你可以通过以下方式验证平衡策略的效果：

1. **混淆矩阵分析**：查看N3和N4行/列的值，验证识别改善情况
2. **每类准确率比较**：与基线模型比较每个睡眠阶段的准确率变化
3. **F1分数提升**：特别关注宏平均F1分数的提升
4. **临床相关指标**：例如睡眠分期的连续性、周期性等临床指标是否更接近真实情况

## 额外建议

1. **考虑N3/N4合并**：在一些睡眠分析标准中，N3和N4已合并为单一的深睡期(N3)
2. **尝试不同alpha值**：如果效果不佳，可尝试调整balance_alpha (0.3-0.7范围)
3. **集成模型方法**：考虑训练多个模型并采用集成方法，进一步提高少数类识别率
